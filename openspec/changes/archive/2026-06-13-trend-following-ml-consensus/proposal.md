## Why

The current legacy architecture relies on unconstrained signals that are highly vulnerable to lookahead bias and lack a unified bounded scale. To build a robust trend-following system that mathematically maps to our known ground truth targets (`isp-regimes-btcusd.csv` and `isp-signals-btcusd.csv`), we need to transition to a normalized probability/intensity space [0,1].

**Mathematical/Statistical Motivation:**
By normalizing all technical and on-chain indicators into a strict continuous [0,1] bounds using causal rolling windows (e.g., 800-day minimum lookback), we eliminate the risk of out-of-sample blowups. This allows the system to aggregate indicator intensity coherently via an additive average consensus model. We formulate this as an ML optimization problem (e.g., L1-Lasso or Logistic Regression) that discovers the optimal constituent weights (and implicitly feature-selects via pruning) to perfectly recreate the historical regime state shifts. This enforces the rule that no single indicator drives the system, and explicitly anchors the output to empirically validated regime shifts.

## What Changes

This change transforms the system into an ML-driven, bounded consensus framework. It impacts multiple architectural layers:

- **Layer 2 (Signal Engine):** All indicator outputs are transitioned from discrete {-1, +1} signals to continuous [0,1] intensity scores. 
- **Layer 3 (Feature Processing):** We introduce rolling causal normalization (Empirical CDF or MinMax over past N days) to ensure no lookahead bias when mapping unbounded signals (like RSI or NUPL) into the [0,1] space.
- **Layer 4 (Ensemble Aggregation):** The aggregation engine is updated to use an ML solver (e.g., L1-regularized linear model) to calculate a weighted average consensus score. This continuous final score [0,1] is then discretized by dynamically tuned boundaries (e.g., >0.8 for Strong Bull) to match the target regime states.

**Backtest Impact:**
By constraining the model through L1 regularization and strictly bounding inputs, the model will inherently drop redundant indicators (enforcing low multicollinearity / VIF < 10). The Sharpe ratio should align closely with the ISP targets, while Max Drawdown will be strictly controlled during regime shifts. Walk-forward validation will be required to ensure the weights generated by the ML solver do not overfit to specific cycles.

**Data Dependencies:**
No new data dependencies are introduced. The system will continue to utilize the OHLCV price series and the existing Glassnode/BRK on-chain metrics (`sth_mvrv`, `sth_nupl`, `sth_sopr_24h`, `sth_supply_in_profit`).

## Capabilities

### New Capabilities
- `ml-consensus-engine`: A machine learning-driven ensemble aggregator that maps normalized indicator intensities into bounded [0,1] final scores and discretizes them into 5 regime states (Strong Bull, Weak Bull, Neutral, Weak Bear, Strong Bear).
- `rolling-normalization`: A data processing utility that safely scales unbounded metrics into [0,1] using strictly causal, lookahead-free rolling windows (800-1200 days).

### Modified Capabilities
- `indicator-signals`: Changes the fundamental contract of all Layer 2 indicators to output [0,1] continuous values rather than discrete [-1, 1] classifications.

## Impact

- `src/signals/`: All indicators will be rewritten to return [0,1] scores.
- `src/features/`: Addition of rolling normalization pipelines.
- `src/ensemble/`: Replacement of the discrete voting engine with an ML-trained weighted average model.
