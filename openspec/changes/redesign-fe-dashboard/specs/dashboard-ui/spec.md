## ADDED Requirements

### Requirement: Premium Dark-Mode Design System Integration
The frontend MUST utilize a design system generated by the Stitch MCP to provide a premium, high-contrast dark mode dashboard suitable for quantitative trading monitoring.

#### Scenario: Rendering the main dashboard
- **GIVEN** the user opens the frontend application
- **WHEN** the dashboard layout is mounted
- **THEN** the UI MUST render using the Stitch-generated typography and color tokens, displaying a dark-mode first interface without generic styling.

### Requirement: HMM Regime Widget
The dashboard MUST feature a dedicated widget displaying the current HMM Regime (Bull, Bear, Sideways) along with its posterior probability.

#### Scenario: Displaying the current regime
- **GIVEN** the backend API returns a regime state with P(Bull) > 0.70
- **WHEN** the Regime widget receives the data
- **THEN** the widget MUST explicitly display "BULL" with a visual indicator (e.g., green highlight) and show the exact posterior probability P(Bull) > 0.70.

### Requirement: Final LTTD Score Gauge
The dashboard MUST present the Final Score (the LTTD output) ranging from -1.0 to +1.0 using a clear visual gauge or dial.

#### Scenario: Rendering the Final Score
- **GIVEN** the ensemble aggregation outputs a Final Score of +0.85
- **WHEN** the Score Gauge component updates
- **THEN** the gauge MUST point to the positive extreme, visually conveying strong bullish momentum in the 120-350 day horizon.

### Requirement: Feature Heatmap / Orthogonalized Indicators Breakdown
The dashboard MUST display a breakdown of the underlying indicators and their values after PCA Orthogonalization.

#### Scenario: Inspecting feature contributions
- **GIVEN** the backend provides the individual orthogonalized Indicator Scores
- **WHEN** the user views the Feature Breakdown section
- **THEN** the UI MUST display a bento-grid or heatmap showing each score, ensuring the operator can see the disaggregated components of the Final Score.

## Non-Goals
- Real-time websocket streaming (daily polling is sufficient).
- Modifying backend endpoints or database structures.
- Allowing manual trade execution from the dashboard (this is a monitoring tool).
